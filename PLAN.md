# PLAN.md — 실시간 전력수급 이상탐지 모니터링 시스템

> 한국 전력계통을 **제조 생산라인**에 빗대어, 5분 단위 실시간 전력수급 데이터를
> 자동 수집하고 **다층 이상탐지(통계·ML·딥러닝)**로 분석하는 무중단·무비용 모니터링 시스템.
>
> (자매 프로젝트: `chungbuk-air-quality-monitor` — SPC 중심 / 본 프로젝트 — 이상탐지 중심으로 차별화)

---

## 1. 목표 (Goal)

- **무엇을:** 전국 전력수급(현재수요·공급예비율·발전믹스·기온)을 실시간 모니터링하고 이상 상황을 탐지·경보
- **왜:** 반도체 팹 등 대규모 제조설비는 전력 안정성이 수율과 직결 → "유틸리티 설비 실시간 감시"를 실데이터로 구현해 **QC/생산관리 역량 + 시계열 이상탐지 역량**을 동시 입증
- **핵심 메시지:** 1번 프로젝트가 *SPC(통계적 공정관리)*였다면, 본 프로젝트는 *ML/DL 기반 이상탐지*로 기법을 차별화

---

## 2. 제조 메타포 (Manufacturing Metaphor)

| 제조 공정 (실제 직무) | 본 프로젝트 |
|---|---|
| 생산라인 부하 / throughput | 현재 전력수요 (부하, MW) |
| 안전재고 · 여유율 | 공급예비율 (%) — 낮으면 블랙아웃 위험 |
| 다중 설비 가동 상태 | 발전원별 발전량 (원자력·LNG·석탄·신재생…) |
| 규격 한계 USL/LSL | 예비율 임계선(경보 기준), 주파수 60Hz |
| 환경 변수 (온습도) | 기온 (수요와 강한 상관) |
| **공정 이상 (Anomaly)** | 수요 급변(스파이크) · 예비율 급락 · 예측오차 폭증 |

---

## 3. 기능 (Features)

1. **실시간 수집** — KPX 5분 단위 전력수급/발전원별 API 폴링 → DB 누적
2. **실시간 대시보드** — 부하 곡선 · 예비율 게이지 · 발전믹스 · 이상 타임라인
3. **다층 이상탐지** (핵심 차별점)
   - L1 통계: EWMA / CUSUM 변화점 탐지
   - L2 ML: Isolation Forest (다변량 이상)
   - L3 DL: LSTM-AutoEncoder (시계열 재구성 잔차)
4. **수요 예측 + 잔차 기반 탐지** — 단기 수요 예측 후 예측오차 이상 감지
5. **경보 알림** — 예비율 임계/이상 감지 시 Discord 웹훅
6. **무중단 운영** — GitHub Actions cron 자동 수집 (컴퓨터 꺼도 누적)

---

## 4. 제외 기능 (Non-goals)

- 전력 가격(SMP) 예측·거래 시뮬레이션 (범위 밖)
- 지역(시도)별 세분화 — 1차는 전국 계통 기준만
- 사용자 로그인 / 다중 사용자 / 권한 관리
- 발전소 단위 제어·예측 (데이터 미공개)

## 5. 작업 순서 (Tasks)

1. **환경/스캐폴딩** — 레포 구조, `.env.example`, `.gitignore`, pyproject, 테스트 골격
2. **수집기** — `collectors/kpx.py` (수급) + 기온 결합, `flows/collect_flow.py`
3. **저장소** — SQLite 스키마(`storage/models.py`), 멱등 upsert
4. **탐지 L1** — `analysis/ewma_cusum.py` (통계 변화점)

6. **탐지 L3** — `analysis/lstm_autoencoder.py` (시계열 잔차)
7. **수요 예측** — `analysis/demand_forecast.py` + 잔차 탐지
8. **대시보드** — Streamlit 멀티페이지 (모니터링/이상/발전믹스/탐지비교)
9. **알림** — `notifier/discord.py` 임계·이상 경보
10. **CI 자동화** — GitHub Actions cron 수집 + 테스트
11. **문서/포트폴리오** — README, 메타포·DMAIC 매핑, 발표 자료

## 7. 이상탐지 설계 (Anomaly Detection Design)

- **정상 패턴:** 전력수요는 강한 일/주 주기성 + 기온 상관 → 단순 임계값은 거짓경보 多
- **접근:** 주기성 제거(예측) → **잔차에 이상탐지** 적용이 핵심
- **3계층 비교 평가:** 동일 이벤트(예: 한파 급증, 예비율 급락)에 L1/L2/L3 성능 비교 → 거짓경보율·탐지지연 표로 정리 (포트폴리오 깊이)
- **라벨 부재 대응:** 비지도 탐지 + 실제 전력경보 발령 이력으로 사후 검증

## 8. 테스트 방법 (Test Plan)

- 수집기: API 응답 파싱 → 스키마/타입 검증, 중복 수집 멱등성
- 탐지기: 합성 시계열(스파이크·계단변화·점진drift) 주입 → 각 탐지기가 잡는지
- 대시보드: `_lib` 데이터 로딩 함수 단위 테스트
- 회귀: `pytest` 전체 통과 유지 (1번 프로젝트처럼 배지화)
